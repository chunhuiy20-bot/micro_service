# A-Level 学习刷题平台 — 教师内容交付说明

> 本文档说明建设 A-Level 智能刷题网站所需的全部内容，以及每项内容对系统的具体用途。
> 平台核心采用 **IRT（项目反应理论）** 自适应评估引擎，能实时估算每位学生的能力值（θ），并根据能力精准推送适配题目。

---

## 一、平台能做什么（给老师的背景说明）

| 系统模块 | 功能描述 |
|---|---|
| **能力估计**（ThetaEstimator） | 每答完一题自动更新学生能力值 θ ∈ [-3, 3]，−3 最弱，3 最强 |
| **题目难度模型**（IRTStrategy） | 根据学生能力 θ 和题目难度 β，预测答对概率，实现精准选题 |
| **难度自动校正**（IRTCalibrator） | 题目积累足够作答数据后，自动修正初始难度，越用越准 |
| **激励积分**（RewardStrategy） | 答对难题得高分，答对易题得低分，无负分，激励学生挑战难题 |
| **遗忘衰减**（ThetaDecay） | 长期不练习时 θ 自动下降，提醒学生保持复习节奏 |

## 二、老师需要提供的内容清单

### 2.1 科目与考纲信息

**每个开设科目需提供：**

```
科目名称         例：Mathematics / Physics / Chemistry / Economics
考试局           Cambridge (CIE) / Edexcel / AQA / OCR（选一）
考试代码         例：9709 / 9702
有效考纲版本     例：2023–2025 syllabus
考试形式         Paper 1 / Paper 2 / Paper 3 各自时长、总分、题型占比
```

**为什么需要：** 系统按考纲划分知识点树，确保题库结构与真实考试对齐。

---

### 2.2 知识点体系（Syllabus Topic Map）

**格式要求：层级结构，最多三级**

```
科目
└── 大章节（Chapter）          例：Pure Mathematics
    └── 子主题（Topic）        例：Functions
        └── 知识点（Learning Point） 例：Composite functions / Inverse functions
```

**每个知识点需标注：**

| 字段 | 说明 | 示例 |
|---|---|---|
| 知识点编号 | 与考纲保持一致 | `1.2.3` |
| 考试局官方描述 | 直接复制考纲原文（英文） | "understand the meaning of..." |
| 常考题型 | MCQ / Short Answer / Structured / Essay | Short Answer |
| 考试权重估计 | 该知识点在全卷的分值占比 | ~8% |
| 前置知识点 | 需先掌握的知识点编号 | `1.1.1`, `1.2.1` |

**为什么需要：** 系统需要知识点图谱来做"知识依赖推题"——学生 Function 没掌握，系统不会推 Calculus 题。

---

### 2.3 题库（最核心内容）

**每道题必须包含以下字段：**

#### 基础信息

| 字段 | 格式 | 说明 |
|---|---|---|
| 题目编号 | `Q-MATH-0001` | 全局唯一，便于追踪 |
| 科目 | 文本 | Mathematics |
| 知识点编号 | 列表 | `["1.2.3", "2.1.1"]`，可多个 |
| 题型 | 枚举 | `MCQ` / `Short Answer` / `Structured` / `Essay` |
| 来源 | 文本 | `Past Paper 2022 P1 Q3` / `原创` |
| 年份 | 数字 | 2022（Past Paper 填写，原创填当年） |

#### 题目内容

| 字段 | 格式 | 说明 |
|---|---|---|
| 题干 | Markdown / LaTeX | 支持公式、图片 |
| 选项 A–D | 文本（MCQ 必填） | 每个选项单独一行 |
| 正确答案 | 文本 | MCQ 填 `A/B/C/D`；主观题填标准答案要点 |
| 评分标准 | Markdown | 主观题必填，说明各步骤得分点 |
| 详解 | Markdown | 完整解题过程，含关键步骤说明 |
| 图片/附件 | 文件路径或 URL | 需要配图的题目 |

#### **IRT 难度参数（重要）**

> 这是系统自适应推题的核心参数，老师需要根据教学经验给出**初始估值**，系统会在学生作答后自动校正。

| 字段 | 范围 | 含义 | 填写建议 |
|---|---|---|---|
| **β（难度）** | `-3.0` 到 `3.0` | 负值=简单，正值=难 | 见下方参考表 |
| **α（区分度）** | `0.5` 到 `2.5` | 越高越能区分强弱学生 | 一般填 `1.0`，考点集中的填 `1.5` |
| **c（猜测率）** | `0.0` 到 `1.0` | MCQ 四选一填 `0.25`，主观题填 `0.0` | 按题型固定填写即可 |

**β 难度参考对照表：**

| β 值 | 难度等级 | 对应 A-Level 场景 |
|---|---|---|
| `-3.0` ~ `-2.0` | 极简单 | 考纲定义类、直接代入公式 |
| `-2.0` ~ `-1.0` | 简单 | AS 级别基础题 |
| `-1.0` ~ `0.0` | 中偏易 | 常规 A-Level 单考点题 |
| `0.0` ~ `1.0` | 中等 | 需要两步推导，A-Level 常规难度 |
| `1.0` ~ `2.0` | 较难 | 多知识点综合，A2 级别 |
| `2.0` ~ `3.0` | 极难 | 历年压轴题、高区分度综合题 |

### 2.4 学习材料（配合刷题使用）

**每个知识点需配套：**

| 材料类型 | 格式要求 | 必须/可选 |
|---|---|---|
| 知识点讲义 | PDF 或 Markdown，含公式、例题 | 必须 |
| 公式速查表 | PDF，按章节整理 | 必须 |
| 重点/易错提示 | 文本列表，每条 ≤ 50 字 | 强烈推荐 |
| 视频讲解链接 | YouTube / 自托管 URL | 可选 |
| 历年真题 PDF | 按年份整理，含 Mark Scheme | 强烈推荐 |

---

### 2.5 学生初始分班信息

系统冷启动时，学生 θ 默认为 `0`（中等水平）。
如果老师能提供**现有班级摸底成绩**，可以让系统从更准确的起点开始：

| 字段 | 说明 |
|---|---|
| 学生 ID | 学校系统内的唯一编号 |
| 科目 | 对应科目 |
| 初始能力估计 | `beginner` / `intermediate` / `advanced`，或直接给摸底考试分数（百分制） |

## 三、文件交付格式规范

### 推荐格式

| 内容类型 | 推荐格式 |
|---|---|
| 知识点体系 | Excel（.xlsx）或 CSV，每行一个知识点 |
| 题库 | Excel（.xlsx），每行一道题，列对应上述字段 |
| 题目图片 | PNG / JPG，文件名与题目编号对应，如 `Q-MATH-0001.png` |
| 学习讲义 | PDF 或 Markdown |
| 学生信息 | Excel（.xlsx）或 CSV |

### 题库 Excel 列头示例

```
题目编号 | 科目 | 知识点编号 | 题型 | 来源 | 年份 | 题干 | 选项A | 选项B | 选项C | 选项D | 正确答案 | 评分标准 | 详解 | β难度 | α区分度 | c猜测率 | 图片文件名
```

## 四、数量建议

| 科目 | 最低题目数（上线） | 目标题目数（良好体验） |
|---|---|---|
| 单科（如 Math） | 300 题 | 1000+ 题 |
| IRT 难度校正生效 | 每题需 ≥ 30 条学生作答记录 | — |
| 知识点覆盖 | 每个知识点 ≥ 10 题 | 每个知识点 ≥ 30 题 |

> **说明：** IRT 难度自动校正（`IRTCalibrator`）需要积累足够的作答数据。上线初期，老师提供的 β 初始估值越准确，系统冷启动体验越好。

## 五、常见问题

**Q：老师不懂 β 值怎么填？**
A：只需填写`极简单 / 简单 / 中等 / 较难 / 极难`五个等级，开发团队统一映射为对应数值。

**Q：Past Paper 题目有版权问题吗？**
A：建议咨询考试局授权政策，或只使用考试局明确允许的题目。原创题目不受此限制。

**Q：题目里有复杂数学公式怎么处理？**
A：平台支持 LaTeX 格式，老师可在题干中使用 `$...$` 包裹公式，如 `$\int_0^1 x^2 dx$`。

**Q：图片怎么提交？**
A：将图片命名为对应题目编号（如 `Q-MATH-0001.png`），与题库文件放在同一个压缩包里提交。

## 六、交付检查清单

老师在提交内容前，请确认以下各项：

- [ ] 考纲版本已确认，与当前考试年度一致
- [ ] 知识点编号与考纲官方编号对齐
- [ ] 题库中每道题都有：题干、正确答案、详解
- [ ] MCQ 题目的 c 值填写了 `0.25`（四选一）或 `0.2`（五选一）
- [ ] 主观题的 c 值填写了 `0.0`
- [ ] β 值已根据经验填写（或标注难度等级）
- [ ] 图片文件名与题目编号一一对应
- [ ] 讲义已按知识点章节命名整理
- [ ] 学生分班信息中没有出现重复 ID
